# 01. 제품 명세 (Product Spec)

## 1.1 비전

"15초 안에 한 라운드, 1분 안에 세 라운드." 출퇴근길·휴식 시간에 가볍게 즐기는 모바일 친화 다른 그림 찾기 게임.
계정·결제·튜토리얼 없이 **링크만 열면 바로 시작**되는 즉시성.

## 1.2 타겟

- 1차: 모바일 웹에서 짧은 시간 즐길 거리를 찾는 캐주얼 유저 (10대 후반 ~ 40대)
- 2차: 데스크톱 브라우저에서 한 손으로 마우스 클릭만 하고 싶은 유저
- 3차: AI로 만든 이미지의 미묘한 차이를 즐기는 디지털 콘텐츠 애호가

## 1.3 MVP 스코프

### In Scope

- 정적으로 미리 생성된 N세트(최소 30세트)의 다른 그림 콘텐츠
- 한 라운드: 좌/우 두 이미지에서 정해진 차이점(보통 5개) 모두 찾기
- 제한 시간(기본 60초), 남은 시간 게이지
- 차이 발견 시 즉각적인 시각/사운드 피드백
- 라운드 종료 시 결과 화면 (성공/실패 + 다시하기 + 다음 라운드)
- 모바일/데스크톱 반응형
- "오늘의 그림" 1세트와 "랜덤 모드" 진입점
- 누구나 접속 가능, 로그인 0
- 사용자 데이터 0 저장 (단, 익명 집계 메트릭은 가능 — 별도 항목)

### Out of Scope (MVP 이후)

- 회원가입/로그인, 랭킹, 친구 초대
- 다국어 (한국어 단일로 시작)
- 결제, 광고, 쿠폰, 인앱 아이템
- 사용자 제작 콘텐츠(UGC) 업로드
- 실시간 멀티플레이
- 푸시 알림, PWA 오프라인 캐시
- 힌트, 스킵, 보너스 시간 등 메타 게임 요소

## 1.4 비기능 요구사항

| 항목 | 목표 |
|------|------|
| LCP (모바일 4G) | < 2.5s |
| 첫 이미지 표시 | < 1.0s (CDN HIT 가정) |
| 정답 클릭 응답 | < 50ms (클라이언트 검증) |
| 가용성 | Cloudflare 가용성에 종속 (≥ 99.95%) |
| 월 운영 비용 (1만 DAU) | < $30 (이미지는 사전 생성, 런타임 0) |
| 접근성 | WCAG 2.1 AA 부분 (대비, 키보드 포커스) |

## 1.5 성공 지표 (KPI)

런타임에 사용자 식별을 하지 않으므로 *집계* 메트릭에 집중한다. (Cloudflare Web Analytics, 쿠키리스)

- D1: 일일 라운드 플레이 수
- 라운드 완주율 = 완료 / 시작
- 평균 클리어 타임 (성공 라운드 기준)
- 콘텐츠별 난이도 편차 (정답률)
- 신고/오답 클릭 비율 (콘텐츠 품질 지표)

## 1.6 위험 요소

| 위험 | 완화책 |
|------|--------|
| 이미지 생성 비용 폭증 | 사전 생성, 런타임 호출 금지. 배치당 비용 상한 설정 |
| GPT-Image 2가 차이를 너무 크게/작게 만듦 | 휴먼 검수 단계 필수, QA 체크리스트 |
| 좌표 기반 정답 검증을 클라이언트에서 우회 | MVP는 허용 (로그인 없음, 무피해). 향후 서명 토큰 |
| 동일 콘텐츠 반복 노출로 인한 이탈 | 클라이언트 셔플 + 최근 본 N개 회피 (sessionStorage 대신 메모리) |
| 부적절한 이미지 생성 | 프롬프트 화이트리스트, 휴먼 검수, 신고 버튼 |
